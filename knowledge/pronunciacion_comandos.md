# Pronunciación de Comandos Técnicos para TTS — Guía Completa para IA

## Metadatos del Conocimiento
- **Dominio**: Text-to-Speech (TTS) de contenido técnico en español
- **Objetivo**: Pronunciación natural y profesional de comandos, flags, rutas, IPs y símbolos
- **Idioma base**: Español con términos técnicos en inglés fonético
- **Alcance**: Linux, Git, Docker, redes, seguridad, scripting y terminología técnica general

---

## 1. Principios Fundamentales

### Reglas de oro para narración técnica
```
✅ Usar "menos" para todos los flags (-, --)
✅ Leer los números de forma completa y natural
✅ Hacer pausa natural entre componentes de comandos complejos
✅ Mantener ritmo constante y profesional
✅ Subir ligeramente el tono en los comandos clave
✅ Pronunciar las rutas de forma fluida, sin deletrear
✅ Los términos en inglés técnico se pronuncian en fonética española

❌ No pronunciar comandos letra por letra
❌ No decir "guión" para los flags (usar "menos")
❌ No acelerar en flags múltiples
❌ No deletrear palabras completas como "nmap" o "grep"
❌ No omitir pausas en comandos con múltiples opciones
```

### Frases de transición para narración
```
Inicio:        "Ahora ejecutamos..." / "Vamos a correr..." / "Usamos..."
Explicación:   "Este comando hace..." / "La salida muestra..." / "Observen cómo..."
Transición:    "Ahora vamos a..." / "El siguiente paso es..." / "Cambiamos a..."
Resultado:     "El resultado indica..." / "Fíjense que..." / "Veamos qué encontramos."
```

## 12. Patrones de Narración Profesional

### Estructura recomendada para cada comando
```
1. Anunciar qué se va a hacer:
   "Ahora vamos a [acción]..."

2. Pronunciar el comando completo con pausas naturales:
   "[comando] [flags] [argumentos]"

3. Explicar brevemente qué hace:
   "Este comando [explicación]..."

4. (Opcional) Mencionar el resultado esperado:
   "Deberíamos ver / obtener / encontrar..."
```

### Ejemplo de narración completa
```bash
# Comando:
nmap -sV -sC -p- --min-rate 5000 10.10.10.1 -oA full_scan

menos menos min guión rate cinco mil
contra diez punto diez punto diez punto uno

Este escaneo revisará los sesenta y cinco mil puertos disponibles
usando scripts por defecto y detección de versiones de servicios.
Los resultados se guardarán en los tres formatos estándar
con el prefijo full underscore scan.

Veamos qué servicios están corriendo en este servidor."
```
